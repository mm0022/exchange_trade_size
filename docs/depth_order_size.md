# 资金费套利单笔报单量（基于盘口深度）

> 策略：**同一交易所，多现货 + 空永续**（delta 中性，吃资金费）。
> 数据：`data/depth_log.csv`，实时盘口快照，2026-06-01 15:22~23:22 UTC（北京 23:22~次日 07:22），960 次采样 / 30s 一轮 / 32 合约买卖两侧。
> 生成：`analyze_depth.py --bps N`（可复跑）。

## 为什么用盘口深度，而不是逐笔成交 P75

逐笔成交分位数测的是「市场典型成交切片大小」（被散户小单+算法拆单主导，中位很小），**不是「你能一笔下多大」**。能下多大由**盘口在容忍价差内的挂单深度**决定。实测 BTC ±3bps 内可吃 $数百万，逐笔 P75 给的 $268 低估约 200 倍——对执行容量是错的指标。

## 套利的腿与瓶颈

| 动作 | 吃哪侧 | 单笔上限 |
|---|---|---|
| 开仓：买现货 + 卖永续 | 现货 ask + 永续 bid | ≤ min(现货ask, 永续bid) |
| 平仓：卖现货 + 买永续 | 现货 bid + 永续 ask | ≤ min(现货bid, 永续ask) |

**现货腿几乎总是瓶颈**（现货深度 ≪ 永续）→ 套利单笔由现货决定，远小于永续单边深度。

## ±3bps vs ±5bps：放宽容忍度，薄币放大最猛

| 币 | OKX 3→5bps | Bybit 3→5bps |
|---|---|---|
| BTC | $306k→$621k (2.0×) | $197k→$463k (2.4×) |
| DOGE | $26k→$41k (1.6×) | $3k→$7k (2.8×) |
| XRP | ✗→**$26k** | $16k→$28k (1.8×) |
| LINK | $5k→$10k (1.8×) | $2k→$5k (2.4×) |
| SUI | $4.5k→$7k (1.6×) | $0.9k→$1.9k (2.0×) |
| AAVE | $283→$913 (3.2×) | $235→$781 (3.3×) |
| ARB | $72→$325 (**4.5×**) | $106→$343 (3.2×) |
| PEPE | 数据丢 | $271→$2.4k (**8.8×**) |

- **薄币（ARB/AAVE/PEPE）放大 3~9 倍**：贴盘口几乎没量，流动性堆在离 mid 几 bps 外，放宽才解锁主力深度。
- **BTC/主流 ~2 倍**：本就深，放宽近线性小幅增长。
- **XRP-OKX 在 5bps「复活」$26k**：价差 7.7bps 落在 ±5bps 带内，必须容忍 ~5bps 才能做。

## 成本口径：放宽容忍度会等比例抬高执行成本

容忍度 N bps，套利**往返 4 笔成交**（开仓现货+永续、平仓现货+永续），总滑点 ≈ **2N bps**：

| 容忍度 | 往返执行成本 |
|---|---|
| 3bps/腿 | ≈ 6bps |
| 5bps/腿 | ≈ 10bps |

→ 用 5bps 把 ARB 单笔从 $72 拉到 $325，是**用翻倍的执行成本换的**。**只有该币 funding > 往返成本（如 5bps→需 >~10bps/周期）才净赚**。薄币上量前必须逐个核 funding 是否覆盖。

**结论**：BTC/DOGE/LINK/SUI 用 3bps 就够深；薄币（ARB/AAVE/PEPE）和 XRP-OKX 要上量得放 5bps，但须确认 funding 盖得住往返 ~10bps。

## 数据口径与已知问题

- **OKX 永续盘口 `size` 是张数**，USDT 深度 = `张 × ctVal × 价格`；`analyze_depth.py` 读入后已 ×ctVal 还原。
- **PEPE-USDT-SWAP 数据丢失**：采集时未乘 ctVal(1e7)、被 CSV 2 位小数四舍五入归零，不可还原。需修 `collect_depth.py`（采集时即乘 ctVal 存真实 USDT）后重采。
- **OKX XRP 永续价差 7.7bps** 异常宽（其它主流 ~1bps），值得单独核实。
- **盘口只有实时快照、无历史**：本结论基于 2026-06-01 当晚深度，非历史窗口；行情变化时应重新采集。
